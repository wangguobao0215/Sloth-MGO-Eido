## 技能定位

Sloth-MGO-Eido 是面向 **B2B SaaS 企业市场部**的 AI 增长运营助手。它采用 1+7 父子架构，由一个父 Skill（指挥官）统一调度七大子模块，覆盖从市场情报采集、内容生产、渠道分发、线索培育到客户增长的完整营销链路。

**核心理念**：
- 替代重复劳动 -- 消除内容撰写、线索清洗、数据汇总、渠道适配中的格式性重复工作
- 连接信息孤岛 -- 让情报、内容、线索、客户、度量之间的数据自动流转
- 沉淀组织智慧 -- 将最佳实践、实验结论、客户之声持续积累进知识库

**产出定位**："增长骨架，等待团队注入灵魂"。逻辑完整但刻意留白，让市场人用自己的判断赋予最终创意和温度。所有输出均为参谋建议，最终决策权永远在市场团队手中。

## 安装与初始化

### 安装

1. 在 QoderWork 中安装 Sloth-MGO-Eido 技能
2. 推荐同时安装以下协同技能以获得最佳体验：

| 技能 | 推荐程度 | 增强点 |
|------|----------|--------|
| Agent-Reach | 强烈推荐 | 17 平台深度搜索，增强情报采集和线索补全 |
| Sloth-Sales-Eido | 推荐 | 市场-销售双向数据桥接 |
| Sloth-SMM-Eido | 推荐 | 自媒体全流程发布 |

### 首次配置

首次对话时，系统会通过 AskUserQuestion 逐步引导完成配置：

1. **企业信息** — 公司名称、产品/服务、目标客户类型
2. **竞品列表** — 3-5 家主要竞品
3. **渠道偏好** — 当前主要推广渠道（公众号/知乎/头条/小红书/LinkedIn/EDM/企微/抖音等）
4. **合规级别** — 标准模式（默认）或严格模式（金融、医疗、教育等受监管行业）
5. **集成配置**（可选）— CRM 对接、企微、飞书

配置信息保存在 `EXTEND.md` 文件中，后续可随时修改。

## 核心功能

### 七大模块概览

| 模块 | 名称 | 触发关键词 | 核心能力 |
|------|------|------------|----------|
| **A** | 内容与创意引擎 | 写文章、文案、白皮书、案例、邮件模板、内容日历、品牌定位、StoryBrand | 内容创作全流程：价值验证 > 竞争定位 > 内容撰写 > 合规审查 |
| **B** | 活动与项目指挥 | 活动、发布会、沙龙、Webinar、预算、项目进度 | 智能策划、执行总控、供应商管理、项目监控、活动 ROI |
| **C** | 市场情报中心 | 竞品、行业趋势、市场分析、品类、情报、监控 | 竞品动态、行业趋势、客户需求变迁、品类机会、Battle Card |
| **D** | 线索培育与转化引擎 | 线索、MQL、SQL、打分、培育、销售反馈、SDR | 线索清洗评分、培育序列、销售赋能、双向反馈、输单分析 |
| **E** | 渠道分发与获客 | 发布、分发、公众号、投放、SEO、EDM、SEM、GEO | 多渠道适配、SEO 优化、付费投放素材、ABM、线索回收 |
| **F** | 客户生命周期增长 | 客户成功、留存、Churn、续费、Upsell、NPS | Onboarding、健康度监测、续费扩展、案例征集、NPS 分析 |
| **G** | 增长度量与实验 | 数据、ROI、归因、看板、A/B 测试、CAC、LTV | 北极星看板、归因分析、A/B 实验、ROI 复盘、预算建议 |

### 意图路由与多意图识别

系统根据用户输入自动判断应调用哪个模块。当输入同时匹配多个模块时（如"写篇针对竞品A的对比文章发到公众号"），会按工作链自然顺序调度：

```
C（获取竞品情报）> A（撰写内容）> E（渠道分发）
```

并告知用户执行计划，确认后再逐步执行。

### 三级风控机制

| 级别 | 触发场景 | 校验内容 |
|------|----------|----------|
| L1 轻量 | 日常内容、话术、查询、报告 | 品牌合规（禁用词 + 广告法 + PIPL），自动执行不阻塞 |
| L2 标准 | 涉及预算的策划、付费投放 | L1 + 预算范围校验 + 历史 ROI 参考 |
| L3 完整 | 新渠道投入、重大活动（>10万）、白皮书 | L1 + L2 + CAC/LTV 推演 + 风险评估，需用户确认 |

合规结果标注：`[合规-红]` 必须修改 / `[合规-黄]` 建议修改 / 无标注即合规通过。

### 跨模块上下文传递

模块间数据自动流转，传递时标注来源：

| 典型流转 | 说明 |
|----------|------|
| C > A | 竞品情报注入内容创作，标注 `[来自情报中心]` |
| A > E | 母版文章自动传入渠道适配 |
| E > D | 渠道回收线索带渠道来源标签 |
| D > A | 销售反馈驱动内容方向调整，标注 `[来自销售反馈]` |
| F > A | 客户案例素材流入内容创作，标注 `[来自客户增长]` |
| G > 全部 | ROI 数据和预算建议注入各模块 |

用户可以说"不用之前的信息"来关闭自动传递。

### 输出密度适配

| 模式 | 触发条件 | 输出特征 |
|------|----------|----------|
| 闪电模式 | 输入简短（<50字）或要求"快速""简要" | 核心结论先行，不超过一屏 |
| 标准模式 | 正常输入（默认） | 按模块完整模板执行 |
| 深潜模式 | 要求"详细""深入""全面分析" | 增加推演深度和备选方案 |

随时切换："简要一点" 切闪电模式，"展开讲讲" 切深潜模式。

## 工作流程指南

### 典型场景 1：撰写竞品对比文章

```
你：帮我写一篇针对竞品A的产品对比文章，发到公众号

MGO 执行链路：
  1. Module C -- 获取竞品A最新情报，生成竞品数据卡
  2. Module A -- 基于情报撰写对比文章母版（自动价值验证 + 合规审查）
  3. Module E -- 适配公众号格式，提供 3 个候选标题
```

### 典型场景 2：活动线索清洗

```
你：清洗一下上周活动的线索名单

MGO 执行链路：
  1. Module D -- 清洗去重、四维打分、分级
  2. H 级线索生成情报卡，标注优先跟进
  3. B/C 级线索自动编排培育序列
  4. 提示是否需要查看转化漏斗数据
```

### 典型场景 3：月度增长复盘

```
你：帮我做本月的增长数据复盘

MGO 执行链路：
  1. Module G -- 汇总北极星指标、渠道表现、内容效能
  2. 生成归因分析报告
  3. 输出下月预算分配建议
  4. 标注异常指标并给出改进建议
```

### 典型场景 4：客户流失预警

```
你：最近有客户流失风险吗？

MGO 执行链路：
  1. Module F -- 运行健康度评分模型
  2. 按风险等级分类客户
  3. 高风险客户生成紧急挽回方案
  4. 中风险客户建议主动触达策略
```

### 知识库管理

系统按任务类型自动加载对应知识库：

| 知识库 | 默认路径 | 注入场景 |
|--------|----------|----------|
| 产品文档库 | `00_Shared/01_Product_Data/` | 内容创作、线索话术 |
| 品牌资产库 | `00_Shared/02_Brand_Assets/` | 合规校验、渠道视觉 |
| 竞品情报库 | `00_Shared/03_Competitor_Intel/` | 差异化内容、情报更新 |
| 客户之声库 | `00_Shared/04_Voice_of_Customer/` | 价值验证、培育序列 |
| 最佳实践库 | `00_Shared/05_Best_Practices/` | 所有模块 |
| 行业情报库 | `00_Shared/06_Industry_Intel/` | 趋势分析、活动选题 |
| 增长数据库 | `00_Shared/07_Growth_Metrics/` | 度量分析、风控 |

知识库路径不存在时降级为无知识库模式运行，不报错，但会标注 `[未加载知识库，建议初始化]`。

入库操作需要用户明确触发（说"入库""存到知识库"），系统会在写入前确认。

## 最佳实践

1. **从小场景开始**：首次使用建议从单模块入手（如写一篇文章或清洗一批线索），熟悉后再尝试跨模块联动场景。

2. **认真完成首次配置**：EXTEND.md 中的企业信息、竞品列表和渠道偏好直接影响所有模块的输出质量。信息越准确，产出越贴合业务实际。

3. **善用跨模块联动**：MGO 的最大价值在于模块间数据自动流转。例如用 C 模块采集的竞品情报，直接驱动 A 模块的内容创作，再通过 E 模块分发到各渠道 -- 一句话触发整条链路。

4. **定期更新知识库**：将高质量的客户案例、竞品分析、实验结论入库（说"入库到XX"），让系统越用越聪明。

5. **利用密度适配**：日常快速查询用闪电模式，重要决策前切深潜模式获取完整分析。不需要每次都看完整报告。

6. **注意合规标注**：`[合规-红]` 标注的内容必须修改后才能发布，不要忽略。受监管行业（金融、医疗、教育）建议在首次配置时选择严格合规模式。

7. **活用局部修改**：对输出不满意时，指定修改某处即可，不需要重新生成全文。系统只重写指定部分并标注 `[已根据反馈调整]`。

## 常见问题

### Q1：首次使用需要准备什么？

只需安装技能即可开始。首次对话时系统会引导你完成配置。如果想提前准备，可以整理好以下信息：公司及产品介绍、3-5 家主要竞品名称、当前主要推广渠道。这些信息越详细，系统输出越贴合你的业务。

### Q2：七个模块需要全部用起来吗？

不需要。每个模块独立运行，按需使用即可。大多数团队从内容引擎（A）、市场情报（C）和线索引擎（D）三个模块开始，逐步扩展到其他模块。

### Q3：合规引擎支持哪些法规？

内置支持中国《广告法》和《个人信息保护法》（PIPL）的合规校验，以及品牌用语规范检查。金融、医疗、教育等行业有额外的行业特定规则。详细合规规则见 [compliance-rules.md](compliance-rules.md)。

### Q4：如何与 Sloth-Sales-Eido 协同？

安装 Sloth-Sales-Eido 后，两个技能自动建立双向数据桥接：MGO 的 D 模块生成的线索情报卡和跟进话术会推送到 Sales-Eido 的 CRM 流程；Sales-Eido 收集的销售反馈会回流到 MGO 的内容和情报模块，驱动内容策略调整。详细协同规则见 [skill-integration.md](skill-integration.md)。

### Q5：输出内容的版本管理怎么做？

关键文件（Battle Card、VoC 摘要、竞品分析报告等）更新时，系统会自动将旧版本追加日期后缀保留。如需回滚，说"回滚XX到上一版本"即可。日常内容草稿和临时文件不做版本追踪。
