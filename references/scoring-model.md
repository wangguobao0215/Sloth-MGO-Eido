# 线索评分模型 Lead Scoring Model

> Sloth-MGO-Eido 参考知识库 | 版本 1.0
> 适用范围：B2B SaaS 市场增长运营线索评分与分级管理

---

## 1. 模型总览

### 1.1 评分体系架构

```
总分 = W1 × D1 + W2 × D2 + W3 × D3 + W4 × D4

其中：
  D1 = 企业画像匹配度得分（Firmographic Fit Score）
  D2 = 行为信号得分（Behavioral Signal Score）
  D3 = 需求明确度得分（Intent Clarity Score）
  D4 = 时效性得分（Recency & Timeliness Score）

默认权重：
  W1 = 0.30（企业画像）
  W2 = 0.30（行为信号）
  W3 = 0.25（需求明确度）
  W4 = 0.15（时效性）
```

### 1.2 分级标准速览

| 等级 | 总分范围 | 含义 | SLA要求 |
|------|---------|------|---------|
| H（Hot） | 80-100分 | 高意向，准备购买 | 48小时内生成情报卡，交付销售跟进 |
| A（Active） | 70-79分 | 中高意向，积极探索 | 7天内培育后交付销售 |
| B（Building） | 50-69分 | 有兴趣，需培育 | 进入培育序列，持续培育 |
| C（Cold） | <50分 | 低意向或不匹配 | 存档观察，季度回顾 |

---

## 2. 维度一：企业画像匹配度 Firmographic Fit（D1）

评估线索所属企业与ICP（Ideal Customer Profile）的匹配程度。

### 2.1 评分细则

| 子维度 | 权重(维度内) | 0-20分 | 21-40分 | 41-60分 | 61-80分 | 81-100分 |
|--------|------------|--------|---------|---------|---------|----------|
| 行业匹配 | 30% | 非目标行业，无已验证案例 | 边缘行业，有少量探索 | 次要目标行业，有1-2个成功案例 | 核心目标行业之一，有成功案例 | 核心ICP行业，已有标杆客户 |
| 企业规模 | 25% | 个体/微型（<20人） | 小型（20-99人） | 中小型（100-499人） | 中型（500-2000人） | 中大型/大型（2000+人），ICP甜蜜区 |
| 营收规模 | 15% | <500万 | 500万-2000万 | 2000万-1亿 | 1亿-10亿 | 10亿+，预算充足 |
| 技术栈匹配 | 15% | 技术栈完全不兼容 | 需大量定制才能集成 | 部分兼容，需适配 | 主流技术栈，易集成 | 与我方产品技术栈天然契合 |
| 地域匹配 | 10% | 不服务区域 | 可远程服务但无本地支持 | 有渠道伙伴覆盖 | 有销售区域覆盖 | 核心城市，有本地团队 |
| 决策结构 | 5% | 决策链不明/政府审批流程极长 | 多层级审批，周期6月+ | 3-4层审批，周期3-6月 | 2层审批，周期1-3月 | 扁平化，1-2层审批，<1月 |

### 2.2 维度得分计算

```
D1 = Σ (子维度得分 × 子维度权重)
   = 行业匹配分×0.30 + 企业规模分×0.25 + 营收规模分×0.15
     + 技术栈匹配分×0.15 + 地域匹配分×0.10 + 决策结构分×0.05
```

### 2.3 数据来源

| 子维度 | 首选数据源 | 备选数据源 |
|--------|-----------|-----------|
| 行业匹配 | 企查查/天眼查行业分类 | 官网About页面 |
| 企业规模 | 企查查人员规模 | LinkedIn公司页面 |
| 营收规模 | 年报/公开财务数据 | 企查查工商信息 |
| 技术栈匹配 | 表单自填/SDR调研 | BuiltWith/技术栈探测 |
| 地域匹配 | 注册地址/官网联系地址 | IP归属地 |
| 决策结构 | SDR沟通获取 | 行业经验默认值 |

## 3. 维度二：行为信号得分 Behavioral Signal（D2）

追踪线索在数字触点上的行为，量化其参与深度与购买意图。

### 3.1 行为事件评分表

| 行为事件 | 基础分值 | 频次加成规则 | 上限 |
|----------|---------|-------------|------|
| **高意图行为（High-Intent）** | | | |
| 申请产品演示/试用 | 30分 | 不叠加 | 30分 |
| 提交价格咨询/报价请求 | 28分 | 不叠加 | 28分 |
| 联系销售（在线/电话） | 25分 | 不叠加 | 25分 |
| 参加产品深度Workshop | 20分 | 不叠加 | 20分 |
| **中意图行为（Mid-Intent）** | | | |
| 下载白皮书/电子书 | 10分 | 每增1次+3分 | 20分 |
| 参加线上直播/Webinar | 10分 | 每增1次+5分 | 25分 |
| 参加线下活动/展会 | 15分 | 每增1次+5分 | 25分 |
| 阅读客户案例页面 | 8分 | 每增1个案例+3分 | 20分 |
| 访问定价页面 | 12分 | 7天内重复访问+5分 | 22分 |
| 阅读产品对比页 | 10分 | 不叠加 | 10分 |
| **低意图行为（Low-Intent）** | | | |
| 阅读博客文章 | 3分 | 每增1篇+1分 | 10分 |
| 社交媒体互动（点赞/评论） | 2分 | 每增1次+1分 | 8分 |
| 打开营销邮件 | 2分 | 每增1次+1分 | 8分 |
| 点击邮件链接 | 4分 | 每增1次+1分 | 10分 |
| 访问官网首页 | 2分 | 7天内重复+2分 | 6分 |
| 订阅Newsletter | 5分 | 不叠加 | 5分 |
| **负面行为（Negative Signal）** | | | |
| 退订邮件 | -10分 | 不叠加 | -10分 |
| 标记邮件为垃圾 | -20分 | 不叠加 | -20分 |
| 30天无任何行为 | -5分 | 不叠加 | -5分 |
| 60天无任何行为 | -15分 | 不叠加 | -15分 |
| 明确表示"不感兴趣" | -30分 | 不叠加 | -30分 |

### 3.2 行为得分计算

```
D2_raw = Σ (各行为事件得分，含频次加成，受上限约束)
D2 = max(0, min(100, D2_raw))

说明：
- 多个行为事件得分可叠加
- 负面行为从总分中扣减
- 最终维度得分限制在 [0, 100] 区间
```

### 3.3 行为数据采集要求

| 数据源 | 采集方式 | 关键字段 |
|--------|---------|---------|
| 官网行为 | JS埋点/Google Analytics | pageview, click, form_submit, session_duration |
| 邮件行为 | ESP（邮件服务商）Webhook | open, click, unsubscribe, bounce |
| 活动参与 | 活动平台API/手动录入 | register, attend, no_show |
| 内容下载 | 表单提交+CDN日志 | download_asset, asset_type |
| 社交互动 | 社交平台API | like, comment, share, follow |
| CRM记录 | CRM系统 | call_log, meeting_log, email_log |

## 4. 维度三：需求明确度 Intent Clarity（D3）

评估线索表达的需求与我方解决方案的匹配程度。

### 4.1 评分细则

| 分值区间 | 需求明确度水平 | 判定标准 |
|----------|--------------|---------|
| 0-20分 | 无明确需求 | 仅浏览通用内容，未表达任何业务需求；来自通用渠道（如品牌广告），无行业/功能指向 |
| 21-40分 | 模糊需求 | 表达了宽泛痛点（如"想提高效率"），未指定品类/方案；下载通用行业报告 |
| 41-60分 | 初步明确 | 明确了业务场景（如"需要营销自动化"），正在了解品类；对比多个方案；提出了初步功能需求 |
| 61-80分 | 需求清晰 | 有明确的功能需求清单；指定了预算范围；有时间线（如"Q2要上线"）；已完成内部初步调研 |
| 81-100分 | 需求精准 | 有详细的RFP/需求文档；指定了POC计划；有明确预算审批；决策层已介入；时间线<3个月 |

### 4.2 需求信号采集

| 信号来源 | 采集方式 | 映射规则 |
|----------|---------|---------|
| 表单字段"您的需求" | 自然语言分析 | 关键词匹配→需求等级 |
| SDR/Sales沟通记录 | CRM备注分析 | BANT维度提取 |
| 搜索关键词 | SEM/SEO数据 | 品类词=初步，品牌词=清晰，长尾词=精准 |
| 内容消费路径 | 行为序列分析 | 从教育内容→方案内容→价格内容=需求递进 |
| 直接请求 | 表单/在线咨询 | 演示请求=清晰，报价请求=精准 |

### 4.3 需求关键词映射

```
搜索/表单关键词 → 需求等级映射：

品类通用词（"CRM系统"、"营销自动化"）→ 41-60分
品类+场景词（"B2B线索管理CRM"）→ 61-80分
品牌词（"[产品名]价格"、"[产品名] vs XX"）→ 61-80分
功能精确词（"[产品名]API集成"）→ 81-100分
RFP相关词（"CRM选型评估"、"营销自动化RFP"）→ 81-100分
```

## 5. 维度四：时效性 Recency & Timeliness（D4）

衡量行为的新鲜度和时间相关性，确保评分反映线索当前状态。

### 5.1 时间衰减模型

```
时效性得分 = 基础时效分 × 衰减系数

衰减系数计算：
  decay_factor = e^(-λ × days_since_last_action)

  其中 λ = 0.03（默认衰减率）
  days_since_last_action = 距最近一次有效行为的天数

衰减系数参考值：
  0天（当日行为）：1.00
  7天前：0.81
  14天前：0.66
  21天前：0.53
  30天前：0.41
  45天前：0.26
  60天前：0.17
  90天前：0.07
```

### 5.2 基础时效评分

| 分值区间 | 时效性水平 | 判定标准 |
|----------|-----------|---------|
| 0-20分 | 冷淡 | 最近行为>60天前；无近期互动；邮件打开率连续3期为0 |
| 21-40分 | 低活跃 | 最近行为30-60天前；仅偶发邮件打开；无主动互动 |
| 41-60分 | 中等活跃 | 最近行为14-30天前；有内容消费行为；参加过1次活动 |
| 61-80分 | 高活跃 | 最近行为7-14天前；多次内容消费；多渠道互动 |
| 81-100分 | 极高活跃 | 最近行为<7天；高频互动（每周2+次）；主动联系/申请演示 |

### 5.3 时间敏感加分项

| 场景 | 加分 | 条件 |
|------|------|------|
| 合同到期窗口 | +15分 | 已知竞品合同到期前3个月内 |
| 财年预算周期 | +10分 | 目标企业财年预算编制期（通常Q4） |
| 行业政策窗口 | +10分 | 相关行业新规发布后3个月内 |
| 融资事件 | +10分 | 企业刚完成融资30天内 |
| 高管变动 | +8分 | 目标岗位（如CMO/CIO）新上任90天内 |
| 竞品负面事件 | +8分 | 竞品出现重大负面30天内 |

## 6. 综合评分与分级

### 6.1 总分计算公式

```
Total_Score = W1×D1 + W2×D2 + W3×D3 + W4×D4

默认权重：
  W1（企业画像）= 0.30
  W2（行为信号）= 0.30
  W3（需求明确）= 0.25
  W4（时效性）  = 0.15

总分范围：[0, 100]

计算示例：
  某线索 D1=75, D2=60, D3=70, D4=85
  Total = 0.30×75 + 0.30×60 + 0.25×70 + 0.15×85
        = 22.5 + 18.0 + 17.5 + 12.75
        = 70.75
  → 等级：A（Active）
```

### 6.2 分级标准与SLA详细定义

#### H级（Hot）— 80-100分

| 属性 | 说明 |
|------|------|
| 定义 | 高匹配+高意向+高活跃，具备短期成交可能 |
| 典型特征 | ICP企业 + 申请演示/报价 + 需求明确 + 近7天活跃 |
| SLA | 48小时内生成情报卡，交付销售跟进 |
| 跟进策略 | 优先排入AE日历，48小时内完成需求初访 |
| 失效规则 | 7天无响应→降为A级；14天无响应→降为B级 |
| 通知机制 | 实时推送至Sales负责人企微/钉钉 + CRM任务创建 |

#### A级（Active）— 70-79分

| 属性 | 说明 |
|------|------|
| 定义 | 中高匹配+积极互动+有一定需求，需推进成熟 |
| 典型特征 | 目标行业 + 多次内容互动 + 初步需求表达 + 30天内活跃 |
| SLA | 7天内通过培育序列提升至H，再交付销售 |
| 跟进策略 | SDR电话/邮件触达，确认BANT后转AE |
| 升级条件 | BANT确认≥3项 + 申请演示 → 升为H级 |
| 失效规则 | 14天无响应→降为B级 |

#### B级（Building）— 50-69分

| 属性 | 说明 |
|------|------|
| 定义 | 有一定匹配或兴趣，但需要持续培育 |
| 典型特征 | 目标行业但规模偏小 / 有行为但需求不明 / 30-60天活跃 |
| SLA | 进入培育序列，持续培育 |
| 跟进策略 | 自动化内容培育（EDM序列+内容推送），每2周一次轻触 |
| 升级条件 | 高意图行为（申请演示/访问价格页）→ 自动升为A级 |
| 失效规则 | 60天无互动→降为C级 |

#### C级（Cold）— <50分

| 属性 | 说明 |
|------|------|
| 定义 | 低匹配或低意向，长期培育或放弃 |
| 典型特征 | 非目标行业/规模 / 60天+无活跃 / 明确表示暂无需求 |
| SLA | 存档观察，季度回顾 |
| 跟进策略 | 低频内容推送，等待信号重新激活 |
| 升级条件 | 任一高意图行为 → 自动重评 |
| 清理规则 | 180天无任何互动 → 归档（不删除，保留回溯能力） |

## 7. 校准机制 Calibration

### 7.1 销售反馈闭环

```
反馈采集周期：每周五17:00前
反馈渠道：CRM中线索状态更新 + 月度评分校准会

反馈类型及处理：

[反馈1] 高分线索（H/A）但实际无效（Disqualified）
  ├── 触发条件：H/A级线索标记为"不合格"或"Lost"
  ├── 分析维度：哪个维度评分虚高
  ├── 调整动作：
  │   └── 单次反馈：记录但不调整
  │   └── 3次同类反馈：下调该维度相关子维度权重5%
  │   └── 5次同类反馈：重新审视该子维度评分标准

[反馈2] 低分线索（B/C）但实际成交
  ├── 触发条件：B/C级线索直接进入商机或成交
  ├── 分析维度：哪个维度评分偏低
  ├── 调整动作：
  │   └── 单次反馈：记录但不调整
  │   └── 3次同类反馈：上调该维度相关子维度权重5%

[反馈3] 跟进时效反馈
  ├── 触发条件：Sales反映SLA不合理（太紧/太松）
  ├── 调整动作：按季度调整SLA标准
```

### 7.2 权重自动微调规则

```
月度校准公式：

W_new(i) = W_old(i) + α × (conversion_rate(i) - avg_conversion_rate)

其中：
  W_new(i) = 维度i的新权重
  W_old(i) = 维度i的旧权重
  α = 学习率，默认0.02
  conversion_rate(i) = 该维度高分线索的实际转化率
  avg_conversion_rate = 全部维度的平均转化率

约束条件：
  - 单次调整幅度 ≤ 0.05（即5%）
  - Σ W(i) = 1.00（归一化）
  - 任意 W(i) ≥ 0.10（下限保护）
  - 任意 W(i) ≤ 0.40（上限保护）
```

### 7.3 校准会议机制

| 频率 | 参与者 | 议程 |
|------|--------|------|
| 周度 | MOps + SDR Lead | 审查H/A级线索跟进情况，快速修正明显误判 |
| 月度 | MOps + Sales Lead + Rev Ops | 分析评分准确率，讨论权重调整，审查反馈 |
| 季度 | CMO + CSO + MOps + Rev Ops | 战略级校准：ICP更新、分级标准调整、SLA优化 |

## 8. 评分引擎运行规则

### 8.1 评分触发机制

| 触发事件 | 重新评分范围 | 频率 |
|----------|-------------|------|
| 新线索入库 | 仅该线索 | 实时 |
| 行为事件发生 | 仅该线索 | 实时（延迟<5分钟） |
| SDR/Sales更新CRM | 仅该线索 | 实时 |
| 时间衰减 | 全库 | 每日凌晨2:00批量 |
| 权重调整 | 全库 | 调整后24小时内批量重算 |
| ICP定义变更 | 全库 | 变更后48小时内批量重算 |

### 8.2 异常处理规则

| 异常场景 | 处理方式 |
|----------|---------|
| 数据缺失（某维度无数据） | 该维度默认给30分，标记"数据待补充" |
| 单行为异常高分（如机器人） | 同一行为10分钟内重复5次→标记异常，人工审核 |
| 企业信息矛盾 | 多个数据源冲突→取最保守值，标记待核实 |
| 评分骤变（24h内变化>30分） | 触发人工审核通知 |
