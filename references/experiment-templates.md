# A/B 实验模板 Experiment Templates

> Sloth-MGO-Eido 参考知识库 | 版本 1.0
> 适用范围：B2B SaaS 市场增长运营实验设计与分析

---

## 1. 实验方法论概述

### 1.1 实验流程

```
假设定义 → 方案设计 → 样本量计算 → 实验执行 → 数据收集 → 结果分析 → 结论记录
```

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 单变量原则 | 每次实验仅改变1个变量，确保因果关系清晰 |
| 随机分组 | 流量/用户随机分入对照组和实验组，消除选择偏差 |
| 统计显著性 | 结果必须达到 p < 0.05 才可判定为有效 |
| 最小可检测效应 | 实验前定义MDE（Minimum Detectable Effect），通常设为10-20% |
| 充分样本 | 样本量必须满足统计功效（Power ≥ 0.80） |
| 预注册 | 实验开始前记录假设和方案，防止事后合理化（HARKing） |

## 2. 假设定义模板 Hypothesis Template

### 2.1 标准假设格式

```
假设编号：HYP-[YYYY][MM]-[序号]
假设日期：[YYYY-MM-DD]
提出人：[姓名/角色]

背景陈述：
  当前[指标名]为[当前值]。我们观察到[现象/问题描述]。
  基于[数据/理论/行业基准]，我们认为有优化空间。

假设陈述（IF-THEN格式）：
  如果我们 [具体改变内容]（自变量），
  那么 [目标指标]（因变量）将会 [预期变化方向及幅度]，
  因为 [原因/机制解释]。

量化预期：
  - 当前基准值：[指标] = [数值]
  - 预期改善幅度：[+XX% / -XX% / +XX个百分点]
  - 最小可检测效应（MDE）：[XX%]

主要指标（Primary Metric）：
  [1个核心指标，用于判定实验成败]

辅助指标（Secondary Metrics）：
  1. [辅助指标1]
  2. [辅助指标2]

护栏指标（Guardrail Metrics）：
  [不应恶化的指标，如退订率不应上升超过X%]

优先级：[P0-关键 / P1-重要 / P2-探索]
预估影响：[高/中/低] × 预估实施难度：[高/中/低]
```

### 2.2 假设示例

```
假设编号：HYP-202401-003
假设日期：2024-01-15
提出人：市场运营经理

背景陈述：
  当前官网Demo申请页的转化率为2.3%。行业基准为3.5%-5.0%。
  通过热力图分析发现，60%的用户在未滚动至CTA区域即离开。

假设陈述：
  如果我们将CTA按钮从页面底部移至首屏可见区域，
  那么Demo申请转化率将提升30%以上（从2.3%到3.0%+），
  因为减少了用户到达CTA的路径摩擦。

量化预期：
  - 当前基准值：转化率 = 2.3%
  - 预期改善幅度：+30%（即+0.7个百分点）
  - MDE：20%

主要指标：Demo申请表单提交转化率
辅助指标：页面平均停留时长、滚动深度
护栏指标：表单完成质量（有效线索比例不低于当前85%）

优先级：P0 | 预估影响：高 × 实施难度：低
```

### 3.2 样本量计算

#### 计算公式

```
适用于转化率类指标的样本量计算（双侧检验）：

n = (Zα/2 + Zβ)² × [p1(1-p1) + p2(1-p2)] / (p2 - p1)²

其中：
  n = 每组所需最小样本量
  Zα/2 = 显著性水平对应Z值（α=0.05 → Zα/2=1.96）
  Zβ = 统计功效对应Z值（β=0.80 → Zβ=0.84）
  p1 = 对照组预期转化率（当前基线）
  p2 = 实验组预期转化率（p1 × (1+MDE)）
  MDE = 最小可检测效应

简化计算表（α=0.05, Power=0.80）：
```

| 当前转化率(p1) | MDE=10% | MDE=15% | MDE=20% | MDE=30% |
|---------------|---------|---------|---------|---------|
| 1% | 158,000 | 70,500 | 39,600 | 17,600 |
| 2% | 77,600 | 34,500 | 19,400 | 8,700 |
| 3% | 51,000 | 22,700 | 12,800 | 5,700 |
| 5% | 29,800 | 13,300 | 7,500 | 3,300 |
| 10% | 14,300 | 6,400 | 3,600 | 1,600 |
| 15% | 9,000 | 4,000 | 2,300 | 1,000 |
| 20% | 6,400 | 2,900 | 1,600 | 720 |

```
使用示例：
  当前转化率 = 2%，期望检测到20%的提升
  查表：每组需要 19,400 个样本
  总样本量 = 19,400 × 2 = 38,800
  日均流量 = 1,000 → 需运行约 39 天
```

#### 实验周期底线

```
最短运行规则：
- 即使样本量提前达标，最少运行7天（覆盖工作日+周末差异）
- B2B场景建议最少运行14天（覆盖2个完整工作周）
- 如涉及邮件类实验，最少运行一个完整序列周期
```

## 4. 四种常见实验场景

### 4.1 场景一：标题测试（Headline Testing）

```
实验名称：[页面/内容] 标题A/B测试
适用渠道：官网着陆页、博客文章、白皮书标题、EDM主题行

对照组（A）：当前标题
  "[当前标题全文]"

实验组（B）：变体标题
  "[变体标题全文]"

变化说明：
  改变了 [措辞/结构/情感调性/数字/长度]
  
主要指标：
  - 着陆页：点击→表单提交转化率
  - 博客：点击率（CTR）
  - 白皮书：下载转化率
  
辅助指标：
  - 页面跳出率
  - 平均阅读时长
  
标题测试最佳实践：
  ├── 一次只改一个维度（如只改数字，或只改情感调性）
  ├── 数字型标题 vs 问题型标题 → 测试吸引力差异
  ├── 利益导向 vs 恐惧导向 → 测试驱动力差异
  └── 短标题(<15字) vs 长标题(15-25字) → 测试完整性偏好
```

### 4.2 场景二：CTA测试（Call-to-Action Testing）

```
实验名称：[页面] CTA A/B测试
适用渠道：官网着陆页、产品页、博客文章CTA、EDM按钮

测试维度（每次选1个）：
  ├── 文案测试：如"免费试用" vs "立即体验" vs "预约演示"
  ├── 颜色测试：如品牌蓝 vs 橙色 vs 绿色
  ├── 位置测试：如首屏 vs 中部 vs 底部 vs 浮动
  ├── 大小测试：如常规按钮 vs 大号按钮
  └── 形式测试：如按钮 vs 文字链接 vs 嵌入式表单

主要指标：
  - CTA点击率（CTR = 点击数 / 页面UV）
  - 表单提交转化率（如CTA指向表单）

辅助指标：
  - 下游转化率（点击CTA后的后续转化）
  - 页面停留时长

CTA测试最佳实践：
  ├── CTA文案使用动词开头（"获取""下载""预约"）
  ├── 按钮颜色与页面主色形成对比
  ├── 首屏CTA + 页底CTA 是最常见高效组合
  ├── 降低摩擦："无需信用卡""30秒注册""免费"
  └── 紧迫性测试："仅剩X个名额"（需真实）
```

### 4.3 场景三：落地页测试（Landing Page Testing）

```
实验名称：[Campaign] 落地页A/B测试
适用渠道：SEM落地页、广告落地页、活动注册页

测试维度（每次选1个）：
  ├── 价值主张文案（首屏headline + subheadline）
  ├── 页面布局（长页面 vs 短页面）
  ├── 社会证明类型（客户Logo vs 数据 vs 证言）
  ├── 表单长度（3字段 vs 5字段 vs 7字段）
  ├── 视觉元素（产品截图 vs 概念插画 vs 视频）
  └── 首屏设计（纯文字+CTA vs 图文并排 vs 视频背景）

主要指标：
  - 表单提交转化率（提交数 / 着陆UV）

辅助指标：
  - 跳出率（Bounce Rate）
  - 表单开始填写率（表单交互数 / 着陆UV）
  - 表单完成率（提交数 / 表单交互数）
  - 线索质量分（提交线索的平均评分）

护栏指标：
  - 线索质量不应下降（有效线索比例 ≥ 当前值-5%）

落地页测试最佳实践：
  ├── 表单字段每减少1个，转化率可提升约5-10%
  ├── 有视频的落地页转化率通常高20-80%
  ├── 移动端优先设计（60%+流量来自移动端）
  ├── 页面加载速度每增1秒，转化率下降约7%
  └── 长页面适合高价产品（需更多信息），短页面适合低门槛CTA
```

### 4.4 场景四：邮件主题行测试（Email Subject Line Testing）

```
实验名称：[邮件Campaign] 主题行A/B测试
适用渠道：EDM营销邮件、培育序列邮件、活动邀请邮件

测试维度（每次选1个）：
  ├── 措辞风格：正式 vs 口语化
  ├── 个性化：含{{FirstName}} vs 不含
  ├── 长度：短主题行(<15字) vs 长主题行(15-30字)
  ├── 内容角度：利益型 vs 好奇型 vs 数据型 vs 紧迫型
  ├── emoji：含emoji vs 不含emoji
  └── 提问 vs 陈述："你知道...?" vs "3个方法..."

执行流程：
  1. 从总发送列表中随机抽取20%作为测试样本
  2. 测试样本平均分为A/B两组（各10%）
  3. 同时发送A/B版本
  4. 等待4小时，统计打开率
  5. 胜出版本发送给剩余80%

主要指标：
  - 打开率（Open Rate = 打开数 / 送达数）

辅助指标：
  - 点击率（CTR = 点击数 / 打开数）
  - 退订率（Unsubscribe Rate）

邮件测试最佳实践：
  ├── 最小测试样本：每组≥500封（统计可靠性）
  ├── 发送时间固定（避免时间变量干扰）
  ├── 个性化主题行通常提升10-20%打开率
  ├── 含数字的主题行通常表现更好
  ├── emoji在B2B场景需谨慎（测试验证，不要假设）
  └── 避免垃圾触发词（"免费""促销""恭喜"等）
```

## 5. 结果分析模板 Result Analysis Template

### 5.1 统计显著性检验

```
检验方法：双侧Z检验（用于转化率类指标）

输入数据：
  对照组：样本量 = nA，转化数 = xA，转化率 = pA = xA/nA
  实验组：样本量 = nB，转化数 = xB，转化率 = pB = xB/nB

计算步骤：

1. 合并转化率：
   p_pool = (xA + xB) / (nA + nB)

2. 标准误差：
   SE = sqrt(p_pool × (1-p_pool) × (1/nA + 1/nB))

3. Z统计量：
   Z = (pB - pA) / SE

4. p-value：
   p = 2 × (1 - Φ(|Z|))    （双侧检验）

5. 判定规则：
   p < 0.05 → 统计显著，拒绝零假设
   p < 0.01 → 高度显著
   p ≥ 0.05 → 不显著，无法拒绝零假设

6. 置信区间（95%）：
   CI = (pB - pA) ± 1.96 × sqrt(pA(1-pA)/nA + pB(1-pB)/nB)
```

## 7. 常见陷阱与注意事项

| 陷阱 | 说明 | 规避方法 |
|------|------|---------|
| 偷看结果（Peeking） | 实验未达样本量就下结论 | 预设样本量，达标前不做判定 |
| 多重比较（Multiple Testing） | 同时测N个变体不做校正 | A/B/n测试使用Bonferroni校正（α/n） |
| 辛普森悖论（Simpson's Paradox） | 总体趋势与分群趋势相反 | 关键维度做分群分析 |
| 新奇效应（Novelty Effect） | 新方案初期高于真实效果 | 实验运行≥2周再做判断 |
| 季节性干扰 | 节假日/行业淡旺季影响数据 | 避开特殊时期，或延长实验覆盖完整周期 |
| 流量不足 | B2B场景流量有限，难以达到样本量 | 适当放宽MDE至20-30%，或使用贝叶斯方法 |
| 事后合理化（HARKing） | 实验失败后重新定义"成功指标" | 预注册假设和主要指标 |
