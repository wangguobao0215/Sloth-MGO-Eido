## 功能概览

Sloth-MGO-Eido 采用 **1+7 父子架构**，由一个父 Skill（指挥官）统一调度七大子模块：

| 模块 | 名称 | 核心能力 |
|------|------|----------|
| **A** | 内容与创意引擎 | 产品文章、白皮书、客户案例、对比文章、EDM 模板，内置价值验证与竞争定位 |
| **B** | 活动与项目指挥 | 活动策划、执行总控表、预算管理、供应商对比、跨模块项目进度监控 |
| **C** | 市场情报中心 | 竞品动态监控、行业趋势追踪、客户需求变迁分析、品类机会发现、Battle Card 生成 |
| **D** | 线索培育与转化引擎 | 线索清洗打分、培育序列编排、销售赋能、市场-销售双向反馈、输单分析 |
| **E** | 渠道分发与获客 | 多渠道内容适配（公众号/知乎/头条/小红书/LinkedIn/EDM）、SEO 优化、ABM 支持 |
| **F** | 客户生命周期增长 | Onboarding 支持、健康度监测、续费/扩展营销、案例征集、NPS 分析 |
| **G** | 增长度量与实验 | 北极星指标看板、归因分析、A/B 实验框架、ROI 复盘、预算建议 |

核心特性：

- **增长飞轮模型** -- 老客户口碑和案例反哺新客户获取，每轮循环积累势能
- **三级风控机制** -- L1 轻量/L2 标准/L3 完整，分级校验避免日常操作阻塞
- **内置合规引擎** -- 广告法、PIPL、品牌规范、行业特定规则多层合规校验
- **混合知识适配器** -- 本地文件夹 + 知识库 API + 向量数据库三级检索
- **跨模块上下文传递** -- 模块间数据自动流转，情报驱动内容，内容驱动分发

## 快速开始

### 安装

1. 在 QoderWork 中安装 Sloth-MGO-Eido 技能
2. 首次对话时系统会自动引导你完成配置（企业信息、竞品列表、模块偏好等）
3. 也可以提前编辑 `EXTEND.md` 文件填写个性化配置

### 最小可运行示例

```
用户：帮我写一篇针对竞品A的产品对比文章

MGO 执行链路：
  1. 模块 C（市场情报）→ 获取竞品A最新情报
  2. 模块 A（内容引擎）→ 撰写对比文章母版
  3. 提示用户是否需要模块 E（渠道分发）适配到各平台
```

```
用户：清洗一下上周活动的线索名单

MGO 执行链路：
  1. 模块 D（线索引擎）→ 清洗、去重、打分、分级
  2. H 级线索标注优先跟进，生成线索情报卡
  3. B/C 级线索自动编排培育序列
```

## 系统架构

```
┌──────────────────────────────────────────────────────────┐
|              Sloth-MGO-Eido（父Skill -- 指挥官）           |
|                                                          |
|   意图识别 -> 风控分级 -> 知识库调度 -> 模块路由 -> 结果汇聚  |
└────┬─────┬─────┬──────┬──────┬──────┬──────┬─────────────┘
     |     |     |      |      |      |      |
     v     v     v      v      v      v      v
   ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
   | A | | B | | C | | D | | E | | F | | G |
   |内容| |活动| |情报| |线索| |渠道| |客户| |度量|
   |引擎| |指挥| |中心| |引擎| |分发| |增长| |实验|
   └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘
```

- **父 Skill**：唯一交互入口，负责意图识别、风控校验、知识库注入、模块调度和跨模块上下文传递
- **子模块 A-G**：无状态执行器，各自拥有独立角色定义、工作流和输出规范
